# [HW1] Обнаружение стен и пола для мобильного робота

Этот репозиторий содержит реализацию двух базовых алгоритмов компьютерного зрения и обработки данных с лидаров, необходимых для навигации мобильного робота в помещении.

## Обзор

Проект разделен на две части:

1.  **Обнаружение стен (2D LiDAR + Hough Transform)**: Восстановление геометрии комнаты (стен) из 2D-скана лидара с использованием преобразования Хафа.
2.  **Обнаружение пола (3D Point Cloud + RANSAC)**: Выделение плоскости пола из 3D облака точек для определения свободного пространства.

## Часть 1. Обнаружение стен (2D LiDAR + Hough Transform)

### Задача
Найти прямые линии (стены) в облаке точек 2D-лидара и визуализировать их поверх исходных данных.

### Используемые данные
В качестве данных используется синтетически сгенерированное облако точек, представляющее собой замкнутый контур комнаты неправильной формы с добавлением шума.

### Алгоритм и реализация

1.  **Генерация данных**: Функция `make_walls` создает точки вдоль заданного многоугольника (`vertex`), добавляя к ним гауссов шум. Это имитирует реальные измерения лидара.

2.  **Упрощение данных (Ramer-Douglas-Peucker)**: Для повышения эффективности преобразования Хафа и удаления шума, облако точек сначала упрощается с помощью алгоритма Рамера-Дугласа-Пекера (функция `split_and_merge`). Это позволяет выделить ключевые точки, аппроксимирующие форму стен.

3.  **Преобразование Хафа**:
    *   Облако точек преобразуется в бинарное изображение с помощью функции `points_to_image`.
    *   Для этого изображения вычисляется преобразование Хафа с помощью `skimage.transform.hough_line`.
    *   В пространстве Хафа ищутся пики (наиболее вероятные линии) с помощью `hough_line_peaks`.

4.  **Визуализация**: Результат отображается в виде исходных точек лидара, на которые наложены найденные линии стен, а также ключевые точки после упрощения и исходные вершины комнаты.

### Результаты

На графике ниже представлен результат работы алгоритма:
*   Синие точки — исходные "измерения" лидара.
*   Красные линии — найденные стены.
*   Красные треугольники — ключевые точки после упрощения (RDP).
*   Зеленые звездочки — исходные вершины комнаты.

## Часть 2. Обнаружение пола (3D Point Cloud + RANSAC)

### Задача
Выделить плоскость пола из 3D облака точек, чтобы робот мог определить безопасную зону для движения.

### Используемые данные
Для работы используется облако точек `fragment.pcd` из стандартной библиотеки Open3D, которое затем прореживается для ускорения вычислений.

### Алгоритм и реализация

1.  **Загрузка данных**: Облако точек загружается из файла примера Open3D и прореживается (`uniform_down_sample`).

2.  **Алгоритм RANSAC (Random Sample Consensus)**:
    *   Реализована функция `ransac_plane`, которая на итеративной основе строит гипотезы о положении плоскости.
    *   На каждой итерации выбираются 3 случайные точки, по которым строится уравнение плоскости.
    *   Рассчитывается расстояние от всех точек до этой плоскости. Точки, расстояние которых меньше заданного порога (`distance_threshold`), считаются "инлайнерами".
    *   Процесс повторяется (`max_trials` итераций). В конце выбирается плоскость с наибольшим количеством инлайнеров.
    *   Ранняя остановка происходит, если найдено достаточно инлайнеров (`min_inlier_ratio`).

3.  **Разделение точек и визуализация**:
    *   Точки, принадлежащие найденной плоскости (инлайнеры), окрашиваются в зеленый цвет (пол).
    *   Остальные точки (аутлайнеры) окрашиваются в красный цвет (препятствия).
    *   Для наглядности в интерактивном 3D-графике отображается найденная плоскость.

### Результаты

Алгоритм RANSAC успешно находит основную плоскость в облаке точек (в данном примере это горизонтальная поверхность, вероятно, стол), которая выступает в качестве "пола". Найденная плоскость описывается уравнением `0.069x -0.016y -0.997z + 1.144 = 0`, и ей принадлежит около 33% всех точек облака.

Ниже представлена 3D-визуализация результата:
*   **Зеленые точки** — точки, отнесенные к плоскости пола.
*   **Красные точки** — препятствия.
*   **Желтая полупрозрачная плоскость** — математическая модель, найденная алгоритмом.
