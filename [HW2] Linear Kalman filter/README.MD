# [HW2] Оценка пройденного пути (Linear Kalman Filter)

Frolova AI, M25-RO-01

Этот проект посвящен реализации линейного фильтра Калмана для объединения данных GPS и акселерометра смартфона. Цель — получить более точную оценку пройденного расстояния по сравнению с "сырыми" измерениями GPS.

## 📖 Обзор

В рамках работы выполняются следующие шаги:

1.  **📱 Сбор данных** на улице (прогулка/бег по прямой 100–150 м) с записью GPS (широта, долгота) и линейного ускорения (акселерометр без гравитации).
2.  **📥 Загрузка и предобработка** CSV-файла с данными.
3.  **📐 Перевод GPS-координат** в метры относительно стартовой точки с помощью формулы Haversine.
4.  **🔊 Оценка шумов** акселерометра (по статическому участку) и GPS (по паспортным данным или разбросу на месте).
5.  **⚙️ Реализация фильтра Калмана** с вектором состояния `[позиция, скорость]` и управляющим воздействием — ускорением.
6.  **📊 Сравнение результатов** фильтра с сырыми данными GPS на графиках.

## 🗺️ Данные

Для выполнения работы использовалось приложение **Physics Toolbox Sensor Suite** (Android). Данные записывались во время движения по прямой на открытой местности.

**Записанные данные (CSV):**
*   `time` — время в секундах.
*   `Latitude`, `Longitude` — координаты GPS.
*   `ax`, `ay`, `az` — линейное ускорение по осям X, Y, Z (м/с²).

## ⚙️ Алгоритм и реализация

### 1. Загрузка и преобразование данных
*   CSV-файл считывается с разделителем `;` и десятичной запятой `,`.
*   По первым ненулевым GPS-координатам определяется точка старта.
*   Функция `haversine_distance` переводит координаты в расстояние в метрах от старта.

### 2. Оценка шумов
*   **`std_acc` (шум процесса)**: вычисляется как стандартное отклонение модуля ускорения на первых 100 точках (телефон неподвижен).
*   **`std_meas` (шум измерения)**: принимается равным `3.0` м (типичная точность GPS).

### 3. Фильтр Калмана
Реализован класс `KalmanFilter` со следующими параметрами:

*   **Вектор состояния:** `x = [position, velocity]ᵀ`
*   **Матрица перехода:** `F = [[1, dt], [0, 1]]`
*   **Матрица управления:** `B = [[0.5*dt²], [dt]]`
*   **Матрица наблюдения:** `H = [[1, 0]]`
*   **Ковариация шума процесса:** `Q` (зависит от `std_acc` и `dt`)
*   **Ковариация шума измерения:** `R = [[std_meas²]]`

На каждом шаге:
1.  Выполняется предсказание (`predict`) с использованием текущего ускорения.
2.  Если есть GPS-измерение (дистанция > 0), выполняется коррекция (`update`).

### 4. Визуализация
Результаты отображаются на четырех графиках:
*   Сравнение дистанции GPS и оценки Калмана (начальный участок).
*   Оценка скорости.
*   Данные акселерометра (ось X).
*   GPS-трек на карте.

## 📈 Результаты

После обработки записанных данных были получены следующие результаты:

*   **Дистанция по GPS:** 375.7 м
*   **Дистанция по Калману:** 374.8 м
*   **Максимальная скорость:** 2.68 м/с (9.63 км/ч)
*   **Среднеквадратичная ошибка (RMSE):** 0.81 м

На графиках видно, что фильтр Калмана сглаживает шум GPS и дает более гладкую оценку траектории, особенно на начальном участке, где GPS-измерения были менее стабильны.

💡 Выводы
Фильтр Калмана успешно объединяет данные GPS и акселерометра, обеспечивая:
- Сглаживание шума GPS, особенно на участках с нестабильным сигналом;
- Более реалистичную оценку скорости, основанную на динамике ускорения;
- Снижение RMSE по сравнению с использованием только GPS.
